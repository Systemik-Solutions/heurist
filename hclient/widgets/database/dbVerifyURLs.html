Check for invalid URLs in Record headers, text and file fields

Checking speed is restricted to 30 URLs per minute to avoid blocking the server. We recommend limiting the check to 150 checks (5 minutes), which will return a list of the bad URLs encountered to that point as a CSV file and a search showing the records containing them so that you can correct them.

You may then run the function again and it will recheck the bad URLs to ensure they have been corrected, then start up where it left off. Repeat as often as needed to correct the whole database
